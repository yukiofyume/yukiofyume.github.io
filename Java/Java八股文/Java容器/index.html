# ArrayList

# 简介

ArrayList 的底层是 Object[] 数组 ，相当于动态数组。与 Java 中的数组相比，它的容量能动态增长。在添加大量元素前，应用程序可以使用 ensureCapacity 操作来增加 ArrayList 实例的容量。这可以减少递增式再分配的数量。

  • RandomAccess 是一个标记接口，用来标识实现类支持快速随机访问；
  • Cloneable 接口标识可以克隆；
  • java.io.Serializable 表示可以序列化。

# 扩容机制

这里以无参构造函数为例子，也就是不指定初始容量，也就是说此时初始化赋值了一个空数组，当向数组中添加第一个元素的时候，数组被扩容为 10。

当 add 第一个元素的时候，经过 calculateCapacity() 方法后会将 DEFAULT_CAPACITY = 10 赋值给 minCapacity，也就是说此时容量变为了 10。

在调用 ensureCapacityInternal() 的时候一定会调用 ensureExplitCapacity() 方法：

  • 当添加第一个元素的时候， len 为 0，因为执行了 ensureCapacityInternal() 方法，此时 minCapacity = 10，因此会调用 grow() 方法。
  • 当添加 2-10 个元素的时候，minCapacity = 2-10，此时 if 不成立，因此不会调用 grow() 方法。
  • 当添加到第 11 个元素的时候， if 成立，进入 grow() 方法进行扩容。
  • 可以看出，当添加第一个元素的时候，满足第 一个 if，所以此时容量设置为了 10；
  • 在扩容的时候，每次都会通过位运算将容量变为 1.5 倍左右；
  • 如果新容量大于了 MAX_ARRAY_SIZE 就会执行， hugeCapacity() 方法来比较 minCapacity（当前元素数量） 和 MAX_ARRAY_SIZE，如果 minCapacity 大于最大容量，那么新容量就是 Integer.MAX_VALUE，不然就是 Integer.MAX_VALUE - 8（将最大容量设置为这样是因为 JVM 在数组对象头中会设置一些参数）。

# HashMap

# 简介

JDK1.8 之前 HashMap 由 数组 + 链表 组成的，数组是 HashMap 的主体，链表则是主要为了解决哈希冲突而存在的（“拉链法” 解决冲突）。

JDK1.8 之后 HashMap 的组成多了红黑树，下面两个条件同时满足，会执行链表转红黑树操作，以此来加快搜索速度。

  • 链表长度大于阈值（默认为 8）；
  • HashMap 数组长度超过 64。

# 为什么 key 不能为 null？

简单来说这里的 put 没有通过 hash 方法，在实际内部的 putVal 方法中不允许 key 为 null。主要原因是因为 currenthashmap.containskey 无法消除二义性，也就是线程 A get 到 null 之后，再调用 containsKey 方法，此时线程 B 将这个 key 插入进来了，就会导致线程 A 判断出错。

# 区别

# HashMap 与 Hashtable

  1. 线程是否安全： HashMap 是非线程安全的， HashTable 是线程安全的，因为 HashTable 内部的方法基本都经过 synchronized 修饰。（如果你要保证线程安全的话就使用 ConcurrentHashMap 吧！）；
  2. 效率： 因为线程安全的问题， HashMap 要比 HashTable 效率高一点。另外， HashTable 基本被淘汰，不要在代码中使用它；
  3. 对 Null key 和 Null value 的支持： HashMap 可以存储 null 的 key 和 value，但 null 作为键只能有一个，null 作为值可以有多个；HashTable 不允许有 null 键和 null 值，否则会抛出 NullPointerException 。
  4. 初始容量大小和每次扩充容量大小的不同 ： ① 创建时如果不指定容量初始值， Hashtable 默认的初始大小为 11，之后每次扩充，容量变为原来的 2n+1。 HashMap 默认的初始化大小为 16。之后每次扩充，容量变为原来的 2 倍。② 创建时如果给定了容量初始值，那么 Hashtable 会直接使用你给定的大小，而 HashMap 会将其扩充为 2 的幂次方大小（ HashMap 中的 tableSizeFor() 方法保证，下面给出了源代码）。也就是说 HashMap 总是使用 2 的幂作为哈希表的大小。
  5. 底层数据结构： JDK1.8 以后的 HashMap 在解决哈希冲突时有了较大的变化，当链表长度大于阈值（默认为 8）（将链表转换成红黑树前会判断，如果当前数组的长度小于 64，那么会选择先进行数组扩容，而不是转换为红黑树）时，将链表转化为红黑树，以减少搜索时间。Hashtable 没有这样的机制。

# HashMap 和 TreeMap

TreeMap 和 HashMap 都继承自 AbstractMap ，但是需要注意的是 TreeMap 它还实现了 NavigableMap 接口和 SortedMap 接口。

实现 NavigableMap 接口让 TreeMap 有了对集合内元素的搜索的能力。

实现 SortMap 接口让 TreeMap 有了对集合中的元素根据键排序的能力，默认是按 key 的升序排序。

综上，相比于 HashMap 来说 TreeMap 主要多了对集合中的元素根据键排序的能力以及对集合内元素的搜索的能力。
