一、IOC和AOP

1.1.IOC

IOC（invert of control） 就是控制反转，是一种思想而不是具体实现，具体来说就是把对象的创建交由 Spring 框架进行管理。IOC 容器是 Spring 实现 IOC 的载体，IOC 容器实际上就是一个 Map，里面存放着各种对象。DI（Dependency Injection）依赖注入是 IOC 的一种具体实现。

举个例子，我们在使用 dao 层做数据处理的时候可以有多种方式，当我们此时想采用一种新的方式的时候，我们只需要修改相应 dao 接口的实现类，然后将之前实现类的注解去掉不装配，这样就可以在不修改原有代码的基础上去实现新功能，这也就是说为什么 ioc 可以让我们可插拔的去实现我们想要的功能。

1.2.AOP

AOP（Aspect oriented Programming） 是面向切面编程，是 OOP（面向对象编程）的一种延续。OOP 通过继承的方式（Animals->(Dog，Cat) 来减少代码冗余，但是如果父类的不同方法中出现了大量的重复性代码（比如日志、性能监控），通过 OOP 是无法消除冗余代码的。这部分重复的代码，叫作横切逻辑代码，横切代码存在代码重复，与业务代码混杂在一起，代码臃肿，不易维护的问题，这个时候就需要 AOP 将业务代码和横切逻辑代码分离出来。

切 ：指的是横切逻辑，原有业务逻辑代码不动，只能操作横切逻辑代码，所以面向横切逻辑。

面 ：横切逻辑代码往往要影响的是很多个方法，每个方法如同一个点，多个点构成一个面。这里有一个面的概念。

原理

  1. 代理的创建
    • 首先，需要创建代理工厂，代理工厂需要 3 个重要信息：拦截器数组、目标对象接口数组、目标对象；
    • 创建代理工厂时，默认会在拦截器数组尾部再增加一个默认拦截器，用于最终调用目标方法；
    • 当调用 getProxy() 方法的时候，会根据接口数大于 0 条件返回一个代理对象（JDK 或 Cglib）；
    • 创建代理对象时，同时会创建一个外层拦截器，这个拦截器就是 Spring 内核的拦截器，用于控制 AOP 的流程。
  2. 代理的调用
    • 当对代理对象进行调用时，就会触发外层拦截器，外层拦截器根据代理配置信息，创建内层拦截器链。创建的过程中，会根据表达式判断当前拦截器是否匹配这个拦截器。而这个拦截器链设计模式就是责任链模式。当整个链条执行到最后时，就会触发创建代理时那个尾部的默认拦截器，从而调用目标方法，最后返回。

二、常见注解

2.1.@Autowired和@Resource

  1. @Autowired 默认按照 byType 方式进行 bean 匹配，@Resource 默认按照 byName 方式进行bean匹配
  2. @Autowired 是 Spring 的注解，@Resource 是 J2EE 的注解。
