# 2. 后端管理系统与前端的交互

# 2.1. 主页

主页展示有试卷总数、题目总数、答卷总数、答题总数。前端在发送一个请求后通过 webpack proxy 转发到后端 DashboardController 中对应的处理方法 index() ，通过 service 层去调用 dao 层的相应方法去数据库 exam_paper、question、exam_paper_answer、exam_paper_question_customer_answer 中查询出所需要的试卷总数、题目总数、答卷总数、答题总数。

# 2.2. 用户管理

# 学生列表

学生列表会展示 id、用户名、真实姓名、年级、性别、手机号、创建时间、状态、操作信息。前端发送一个请求转发到 UserController 之后通过 service 层去调用 dao 层的相关方法根据 role 字段去数据库中查询，如果 role = 1 就说明是学生，返回相应信息后通过 PageHelper 分页返回到前端并展示。

@RequestBody UserPageRequestVM：userName=？，role == 1。

# 管理员列表

和学生列表一样，因为 /page/list 的 @RequestBody 接收后会转化为 （username，role） 的 VM，这样根据 role 就可查询出该用户是管理员还是学生了，然后将符合要求的查询返回就好了。

# 2.3. 卷题管理

# 题目列表

题目列表主要展示 id、学科、题型、题干、分数、难度、创建时间以及操作 。前端发送请求后会对应后端的 EducationController 和 QuestionController 。 QuestionController 中的 pageList() 方法首先通过 questionService 调用 dao 层根据 VM 中的条件查询 question 表中数据返回给 PageInfo，然后在根据查询出来的 info_text_conten_id 去调用 textContentService 查询出题干。 EducationController 通过 subjectService 调用 dao 层查询 subject 表然后返回信息。这样完成了展示数据的查询。

QuestionPageRequestVM：id、level、subjectId、questionType。 ExamPaperController 中的 pageList() 方法首先通过 examPageService 调用 dao 层根据 VM 去条件查询 exam_paper 表。

# 试卷列表

试卷列表需要展示 id、学科、试卷名称、创建时间。前端发送请求后会对应后端的 ExamPaperController ，然后根据 mapping 找到对应的方法，然后根据传递过来的 vm 去 service 层调用 dao 层去数据库中查询相关信息。同时会有请求到达 EducationController ，通过 service 层调 dao 层然后去数据库中查询出所有学科信息，返回给前端，由前端根据 subject_id 去完成拼接。

# 2.4. 任务管理

# 创建任务

前端发送请求后会对应后端 ExamPaperController ，service 层根据前端传递的 vm 参数去调用 dao 层对数据库进行查询，然后将结果返回。

# 任务列表

前端需要展示的有 id、标题、年级、发送者、创建时间。对应后端 TaskController ，service 层根据前端传递的 vm 参数去调用 dao 层去数据库查询，然后将结果返回。

# 2.5. 教育管理

# 学科列表

前端发送请求后会对应后端的 EducationController 。首先会展示的数据有 id、学科、年级、操作。通过指定的 url 找到对应的响应方法，VM 为 id 以及 年级 level，通过 subjectService 调用 dao 层，根据 level 这个条件从数据库中查询出符合要求的行，然后通过 pagehelper 进行分页。

# 2.6. 成绩管理

前端发送请求后对应后端的 ExamPaperAnswerController 和 EducationController 。展示的数据有 id、试卷名称、用户名称、得分、题目对错、耗时、提交时间。 ExamPaperAnswerController 会根据 ** 学科 id（可以为空）** 进行条件查询 exam_paper_answer、subject，通过 exam_paper_answer 查询出来的 create_user 字段去 user 表中查询相应的用户名称，最后返还给前端，这样就完成了答卷列表的查询与展示。

# 2.7. 消息中心

# 消息列表

前端发送请求后对应后端的 MessageController 。展示的数据有 id、标题、内容、发送人、接收人、已读数、接收人数、创建时间。 pageList() 方法会根据 sendUserName（可以为空） 首先去查询 message 表，然后根据 message_id 去 message_user 中查询出接收人、已读数以及接收人数，然后将这些数据封装为 vm 返回给前端。

# 消息发送

# 2.8. 日志中心

前端发送请求后对应后端的 UserController 。展示的数据有 id、用户名、真实姓名、动态、创建时间。根据 ** 用户 id、用户名（都可以为 null）** 从 user_event_log 中查询数据并返回给前端。

# 分数怎么判断

学生在提交之后首先会去根据 exam_paper_id 去 exam_paper 表中将试卷的 frame_text_conent_id 查询出来，然后根据查询到的 id 去 text_content 表中查询出具体的内容，以 json 形式保存的试卷信息，然后在将每个题目的正确答案和学生所提交的答案做对比，就可以得到每题的成绩，最后汇总保存到 question_customer_answer 表中。

# 登录怎么做的

通过自定义配置类继承 WebSecurityConfigurerAdapter 重写 configure() 方法， http.addFilterAt(authenticationFilter(), UsernamePasswordAuthenticationFilter.class) ，负责将（这是一系列的自定义过滤链，包括自定义身份验证成功处理程序、自定义身份验证失败、框架提供的 AuthenticationManager 以及自定义 UserDetailsService）注册加入 UsernamePasswordAuthenticationFilter.class ，这样就完成了一个基本的过滤链。用户在登录的时候请求会被拦截，然后通过自定义 AuthenticationProvider 去进行认证工作，根据成功与否完成不同后续动作。授权是根据配置的 .antMatchers.(url).hasRole(role) 去完成的。
