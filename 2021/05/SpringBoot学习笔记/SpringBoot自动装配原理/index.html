SpringBoot自动装配原理

发表于 2021-05-13 更新于 2021-05-21 分类于 SpringBoot学习笔记

1. SpringBoot 自动装配原理？

1.1. 自动装配的核心注解 @SpringBootApplication‘

@SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan

  • @Configuration：允许在上下文中注册额外的bean或者导入其他配置类；
  • @EnableAutoConfiguration：启动 SpringBoot 的自动配置机制；
  • @ComponentScan：扫描被 @Component（@Service，@Controller）注解的 bean，注解会默认扫描启动类所在包下的所有类。

@EnableAutoConfiguration是实现自动装配的核心注解

其核心功能是由 AutoConfigurationImportSelector 类来实现的

从继承和实现关系来看，AutoConfigurationImportSelector 实现了 ImportSelector 接口，也就是实现了 ImportSelector 接口的 selectImports 方法，这个方法主要用于获取所有符合条件的类的全限定类名，这些类需要被加载到 IOC 容器中。

其中 getAutoConfigurationEntry() 方法的调用链如下：

AutoConfigurationImportSelector.AutoConfigurationEntry getAutoConfigurationEntry(AnnotationMetadata annotationMetadata) $\rightarrow$

SpringFactoriesLoader.loadFactoryNames $\rightarrow$

SpringFactoriesLoader.loadSpringFactories(ClassLoader classLoader)

loadSpringFactories 方法下有如下代码：

Enumeration urls = classLoader.getResources("META-INF/spring.factories"); 会从 “META-INF/spring.factories” 中加载自动配置类。

分析 getAutoConfigurationEntry() 源码：

  1. 第一步：判断自动装配是否打开
  2. 第二步：用于获取EnableAutoConfiguration注解中的 exclude 和 excludeName。
  3. 第三步：获取需要自动装配的所有配置类，读取META-INF/spring.factories
  4. 第四步：筛选需要加载的配置

1.2. 总结

  1. SpringBoot 的自动配置是由 @SpringBootApplication 来实现的

  2. 而 @SpringBootApplication 实现了 @Configuration + @EnableAutoConfiguration + @ComponentScan

  3. @EnableAutoConfiguration 注解 调用了 AutoConfigurationImportSelector 类，AutoConfigurationImportSelector 类实现了 ImportSelector 接口，也就是实现了 ImportSelector 接口的 selectImports 方法，这个方法主要用于获取所有符合条件的类的全限定类名，这些类需要被加载到 IOC 容器中。
