# 1. Spring Security 登录验证

类作用
AuthenticationBean定义登录认证的基本参数
LoginAuthenticationEntryPoint未登录的时候，会重定向到登录页面
RestAccessDeniedHandler拒绝访问处理，代表所登录的用户没有权限
RestAuthenticationFailureHandler用户名或密码错误处理类
RestAuthenticationProvider进行认证
RestAuthenticationSuccessHandler登录成功后的一些处理逻辑，主要用来将登录日志保存到 t_user_event_log 表中
RestDetailsServiceImpl验证通过之后，第二、三... 请求，会调用此类
RestLoginAuthenticationFilter将登录参数序列化为字节流
RestLogoutSuccessHandler用户登出处理类
RestTokenBasedRememberMeServices通过 Base-64 编码的 cookie 识别以前记住的用户
RestUtil工具类
SecurityConfigurerSpring Security 的配置类

# 1.1. SpringSecurity 的核心组件：

  • Authentication：存储了认证信息，代表当前登录用户；
  • SecurityContext：上下文对象，用来获取 Authentication ；
  • AuthenticationManager：认证管理器，用来做授权，要么授权成功后，返回认证对象；要么认证失败后，抛出相应的认证异常。
  • SecurityContextHolder：上下文管理对象，用在程序的任何地方获取 SecurityContext

# 1.2. Spring Security 的认证流程：

查询用户数据 判断账号密码是否正确 正确则将用户信息存储到上下文中 上下文中有了这个对象则代表该用户登录了

# 4. 项目部署

采用集成部署的方式。

  1. 将两个前端项目使用 npm run build 进行打包，然后将生成的目录 student 和 admin 放到对应项目的 resource 下

  2. 修改 application-prod.yml 中的 datasource 和 redis 地址，然后将两个后端项目打包成两个 jar 包，移动到服务器上。

  1. 执行命令启动程序 nohup java -Duser.timezone=Asia/Shanghai -jar -Dspring.profiles.active=prod xxx.jar > start1.log 2>&1 &
  2. 管理端的访问为：ip:8060/admin；学生端的访问为：ip:8061/student。
