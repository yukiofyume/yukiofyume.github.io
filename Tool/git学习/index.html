# 初始化库

1.初始化库
git init
2.添加文件到暂存区
git add a.txt
3.从暂存区删除文件，不会影响本地代码
git rm --cached a.txt
4.提交到本地库
git commit -m "first commit" a.txt

查看提交信息 git reflog or git log

# 版本穿梭

  • --soft：仅仅在本地库移动 head 指针
  • --mixed：在本地库、重置暂存区移动 head 指针
  • --soft：在本地库、重置暂存区、重置工作区移动 head 指针

回退到第二个版本

git reset --hard 5bbcdc1

# 分支操作

命令作用
git branch 分支名创建分支
git branch -v查看分支
git checkout 分支名切换分支
git merge 分支名把指定的分支合并到当前分支上

查看分支版本

git branch -v

创建新的分支

git branch slave-fix

git branch -v

切换分支

git checkout slave-fix
git branch -v

修改分支并提交

合并分支

master 分支没有修改，所以合并的时候不会产生冲突。

git checkout master
git merge slave-fix

冲突合并

合并分支时，两个分支在同一个文件有两套完全不同的修改。需要人为去修改

当两个分支都 commit 之后执行合并发生错误

git merge slave-fix

手动去修改冲突的位置，然后

git add a.txt
git commit -m "success merge"

这里还在合并中，所以直接添加然后提交就可以了，并且只有 master 分支改变，slave-fix 分支不会发生变化。

# github

# 基本使用

  1. 为远程库创建别名： git remote add ceshi https://github.com/yukiofyume/git-demo.git
  2. 推送到远端： git push ceshi master 这里没有办法用密码登录了，要去 GitHub 设置 token
  3. 在修改远端内容时，可以拉去下来： git pull ceshi master
  4. 别人在想向你的库推送代码需要你先将别人邀请进行，在对应项目进行设置

# ssh 使用

  1. 生成密钥： ssh-keygen -t rsa -C example@qq.com
  2. win 下将 .pub 复制，添加到 GitHub 上： C:\Users\leewenhao\.ssh
